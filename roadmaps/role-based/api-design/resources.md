# API Design — resources

> roadmap.sh: https://roadmap.sh/api-design

## Books
- **Web API Design: The Missing Link** (Google / Apigee) — concise, practical REST design principles.
- **API Design Patterns** (JJ Geewax) — comprehensive catalog of patterns for robust, evolvable APIs.
- **RESTful Web APIs** (Richardson & Amundsen) — hypermedia, HATEOAS, and resource-oriented design.
- **Designing Web APIs** (Jin, Sahni & Shevat) — pragmatic guide covering auth, pagination, and versioning.

## Courses / practice
- [MDN Web Docs — HTTP](https://developer.mozilla.org/en-US/docs/Web/HTTP) — authoritative reference for HTTP methods, status codes, and headers.
- [OpenAPI Specification](https://spec.openapis.org/) — the canonical spec for describing REST APIs.
- [Microsoft REST API Guidelines](https://github.com/microsoft/api-guidelines) — battle-tested, opinionated design conventions.
- [roadmap.sh API Design](https://roadmap.sh/api-design) — the interactive roadmap with per-node resources.
