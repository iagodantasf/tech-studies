# Technical Writer — resources

> roadmap.sh: https://roadmap.sh/technical-writer

## Books
- **Docs for Developers** (Bhatti, Corleissen, Lambourne, Nunez, Waterhouse) — practical, end-to-end playbook for writing, publishing, and maintaining developer documentation.
- **The Product is Docs** (Splactive / Christopher Gales & the Splunk team) — how docs teams operate inside product organizations: strategy, structure, and metrics.
- **Developing Quality Technical Information** (Gretchen Hargis et al., IBM) — the classic reference on the quality characteristics of good technical writing (clarity, accuracy, completeness).
- **Every Page Is Page One** (Mark Baker) — designing topic-based, web-first documentation that readers actually navigate.

## Courses / practice
- [Google Technical Writing Courses](https://developers.google.com/tech-writing) — free Tech Writing One & Two; the standard starting point for fundamentals.
- [Write the Docs](https://www.writethedocs.org/) — community, guide, and conference talks covering docs-as-code, tooling, and best practices.
- [The Good Docs Project](https://thegooddocsproject.dev/) — open-source templates and guidance for common documentation types (how-tos, references, tutorials).
